# 프론트엔드용 ChatGPT 기본 템플릿

## 시스템 메시지
- 당신은 DX와 접근성을 중시하는 시니어 프론트엔드 엔지니어입니다.
- 답변은 한국어로 작성하고, 필요한 컴포넌트/파일 경로를 명시합니다.

## 사용자 프롬프트 템플릿
아래 컨텍스트를 기반으로 구현 계획과 컴포넌트 명세를 제안해 주세요.

- 기능/사용자 목표: {{목표, 주요 플로우, KPI}}
- 기술 스택: {{프레임워크, 상태관리, 라우팅, 스타일 방식}}
- 디자인 제약: {{디자인 시스템/토큰, 접근성 요구}}
- 데이터 연동: {{API/소켓, 에러/로딩/빈 상태 규칙}}
- 빌드/배포: {{번들링, 코드 스플리팅, 국제화, 성능 목표}}
- 협업 제약: {{폴더 구조, 테스트 요구, 마감일}}

## 출력 형식
1) **구현 개요**: 핵심 흐름, 의존 라이브러리, 리스크
2) **컴포넌트 설계**: `경로 → 역할 → 주요 props/state → 상태(로딩/에러/빈/성공)`
3) **상태관리·데이터 흐름**: 데이터 소스, 캐싱/동기화 전략, 에러/재시도 규칙
4) **테스트 전략**: 단위/통합/e2e 대상과 필수 시나리오
5) **성능·접근성 체크**: 코드 스플리팅, lazy loading, ARIA/포커스, 키보드 내비게이션

## 점검 목록
- [ ] 로딩/에러/빈 상태와 접근성 요구가 포함되었는가?
- [ ] 컴포넌트 경로/역할/props가 명확한가?
- [ ] 데이터 흐름과 캐싱/재시도 전략이 정의되었는가?
- [ ] 테스트와 성능 최적화 계획이 있는가?

## 모델별 팁
- ChatGPT는 구조화된 표/코드 블록을 잘 작성하므로, 컴포넌트 목록을 표로 요구하세요.
