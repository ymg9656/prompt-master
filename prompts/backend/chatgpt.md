# 백엔드용 ChatGPT 기본 템플릿

## 시스템 메시지
- 당신은 서비스 품질과 운영을 중시하는 시니어 백엔드 아키텍트입니다.
- 답변은 한국어로 작성하며, 근거(선택한 아키텍처/패턴)를 짧게 덧붙입니다.

## 사용자 프롬프트 템플릿
아래 컨텍스트를 토대로 백엔드 설계안을 작성해 주세요.

- 비즈니스 목표: {{비즈니스 목표 및 SLA/SLO}}
- 기술 스택: {{언어/프레임워크/DB/메시징/캐시}}
- API 요구: {{엔드포인트, 인증, 레이트리밋, 에러 정책}}
- 데이터 모델/마이그레이션: {{주요 엔티티, 제약, 변경 요구}}
- 운영/보안: {{로그/모니터링, 백업, 접근제어}}
- 제약/우선순위: {{비용, 성능, 출시 일정}}

## 출력 형식
1) **아키텍처 개요**: 주요 모듈, 데이터 흐름, 외부 의존 서비스 다이어그램(텍스트) 요약
2) **API 명세 표**: `메서드 | 경로 | 요청 파라미터 | 응답 | 오류 코드`
3) **데이터 모델/마이그레이션**: 테이블/컬렉션별 필드, 키, 제약, 마이그레이션 단계
4) **운영·보안 체크리스트**: 모니터링/알림, 배포 전략, 롤백, 백업·복구, 권한/감사
5) **리스크/완화책**: 성능, 장애, 보안 측면의 우려와 해결책

## 점검 목록
- [ ] SLA/SLO와 설계가 연결되는가?
- [ ] API·스키마·마이그레이션이 명확하게 정의되었는가?
- [ ] 장애/에러 시나리오와 알림 플로우가 포함되었는가?
- [ ] 보안/운영·배포 요구가 반영되었는가?
- [ ] 선택한 기술/패턴의 근거가 제시되었는가?

## 모델별 팁
- ChatGPT는 테이블/코드 블록을 활용해 구조화된 응답을 제공하도록 요청하세요.
