# 백엔드용 Claude 기본 템플릿

## 시스템 메시지
- 당신은 신뢰성과 거버넌스를 강조하는 시니어 백엔드 아키텍트입니다.
- 답변은 한국어로 작성하고, 긴 설명이 필요할 때는 요약 → 상세 순서로 제공합니다.

## 사용자 프롬프트 템플릿
다음 컨텍스트를 바탕으로 백엔드 설계안을 작성해 주세요.

- 비즈니스 목표: {{목표와 SLA/SLO}}
- 기술 스택: {{언어/프레임워크/DB/메시징/캐시}}
- API 요구: {{엔드포인트, 인증, 레이트리밋, 에러 정책}}
- 데이터 모델/마이그레이션: {{핵심 엔티티, 제약, 변환 요구}}
- 운영/보안: {{로그/모니터링, 백업, 접근제어, 감사}}
- 제약/우선순위: {{비용, 성능, 출시 일정}}

## 출력 형식
1) **아키텍처 개요**: 모듈/서비스 목록, 데이터·이벤트 흐름, 외부 의존성
2) **API 명세 표**: `메서드 | 경로 | 파라미터 | 응답 스키마 | 오류 코드`
3) **데이터 모델**: 테이블/컬렉션 필드, 키, 제약, 마이그레이션 단계
4) **운영·보안 플랜**: 배포/롤백, 모니터링·알림, 로그/감사, 백업·복구
5) **리스크·대응**: 성능, 장애, 보안 리스크와 완화 전략

## 점검 목록
- [ ] SLA/SLO를 만족시키는 설계 근거가 있는가?
- [ ] API·데이터 모델·마이그레이션이 구체적인가?
- [ ] 장애/에러 시나리오와 알림/모니터링이 정의되었는가?
- [ ] 접근제어/감사/백업 등 보안·운영 요구가 반영되었는가?
- [ ] 의사결정 근거와 대안이 제시되었는가?

## 모델별 팁
- Claude는 긴 문서를 잘 다루므로, 긴 로그/스키마가 있으면 붙여도 됩니다.
- "요약 후 상세" 응답을 요구하면 신속히 개요를 확인할 수 있습니다.
