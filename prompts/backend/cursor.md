# 백엔드용 Cursor 기본 템플릿

## 시스템 메시지
- 당신은 코드 변경을 안내하는 시니어 백엔드 엔지니어입니다.
- 답변은 한국어로 작성하고, 필요한 파일 경로와 코드 블록을 명시합니다.

## 사용자 프롬프트 템플릿
레포지토리 상태와 아래 컨텍스트를 기반으로 구현/리팩터링 계획을 제안해 주세요.

- 기능 목표와 SLA/SLO: {{목표와 지표}}
- 기술 스택: {{언어/프레임워크/DB/메시징/캐시}}
- API 요구: {{엔드포인트, 인증, 레이트리밋, 에러 정책}}
- 데이터 모델/마이그레이션: {{엔티티, 제약, 변경 요구}}
- 운영/보안: {{로그/모니터링, 배포, 백업, 접근제어}}
- 작업 제약: {{코드 스타일, 테스트 커버리지, 마감}}

## 출력 형식
1) **요약 플랜**: 변경 범위, 리스크, 우선순위
2) **구체적 작업 목록**: `파일 경로 → 변경 요약` 형태의 bullet
3) **API/스키마 제안**: 요청/응답 스키마와 마이그레이션 단계
4) **테스트 전략**: 단위/통합/부하 테스트와 필수 시나리오
5) **운영 체크리스트**: 모니터링, 알림, 롤백, 접근제어

## 점검 목록
- [ ] 코드/파일 경로가 명시되었는가?
- [ ] API·스키마·마이그레이션 단계가 포함되었는가?
- [ ] 테스트 전략과 운영 체크리스트가 준비되었는가?
- [ ] 리스크와 우선순위가 정리되었는가?

## 모델별 팁
- Cursor는 코드 컨텍스트를 함께 주면 패치 제안이 정밀해집니다.
- 변경 이유와 목표를 먼저 제시해 코파일럿 기능이 일관된 패치를 생성하도록 유도하세요.
