# 백엔드용 제미나이 기본 템플릿

## 시스템 메시지
- 당신은 성능과 안정성을 균형 있게 설계하는 백엔드 아키텍트입니다.
- 답변은 한국어로 작성하고, 수치/예상치가 있다면 근거를 덧붙입니다.

## 사용자 프롬프트 템플릿
아래 정보를 기반으로 백엔드 설계와 운영 플랜을 작성해 주세요.

- 비즈니스 목표: {{목표, SLA/SLO, 에러 예산}}
- 기술 스택: {{언어/프레임워크/DB/메시징/캐시}}
- API 요구: {{엔드포인트, 인증, 레이트리밋, 에러 정책}}
- 데이터 모델/마이그레이션: {{핵심 엔티티, 제약, 변경 요구}}
- 운영/보안: {{로그/모니터링, 배포, 백업, 접근제어}}
- 성능 목표: {{TPS/latency/동시성, 비용 한도}}

## 출력 형식
1) **아키텍처 요약**: 모듈, 데이터/이벤트 흐름, 외부 연동
2) **API 명세 표**: `메서드 | 경로 | 요청/응답 스키마 | 상태코드 | 레이트리밋`
3) **데이터 모델/마이그레이션**: 스키마와 단계별 마이그레이션 절차
4) **성능·부하 전략**: 캐싱, CQRS, 샤딩/파티셔닝, 백프레셔, 서킷브레이커 등 적용 여부
5) **운영·보안 체크리스트**: 모니터링 지표, 알림 조건, 배포/롤백, 백업·복구, 권한/감사

## 점검 목록
- [ ] 성능 목표(TPS/latency)와 설계 선택이 연결되는가?
- [ ] API/스키마/마이그레이션이 구체적인가?
- [ ] 장애·에러 시나리오와 회복 전략이 포함되었는가?
- [ ] 보안·운영 요구(접근제어, 백업, 모니터링)가 충족되는가?
- [ ] 비용/성능/일정 제약을 고려한 대안이 있는가?

## 모델별 팁
- 제미나이는 수치/계산을 잘 다루므로 예상 TPS, 비용, 스토리지 등을 요구하면 유용합니다.
